# Cross-Cutting Test Strategy

- `contract`: API, event schema, and integration provider/consumer tests.
- `e2e`: race-day journeys across frontend, services, workflows, Digital Twin, and AI governance.
- `performance`: load, stress, soak, and failover tests for race-day peak usage.
- `security`: authz, tenant isolation, secret handling, dependency, container, IaC, and DAST checks.
